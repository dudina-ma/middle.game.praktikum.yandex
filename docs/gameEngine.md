# Игровой движок Sea Battle

## Общая архитектура

Движок следует принципам **Flux-архитектуры**:

1. **Unidirectional Data Flow**: Данные текут строго в одном направлении:
   Controller -> Store -> View.
2. **Single Source of Truth**: Всё состояние игры централизовано в Store.
3. **Immutability**: Каждое обновление доски создает новую ссылку на массив,
   что предотвращает побочные эффекты и упрощает отладку.

## Базовые классы

### Game

Основной класс-оркестратор. Управляет циклом рендеринга и слоями анимации.

**Аргументы конструктора:**
| Имя | Описание |
| :--- | :--- |
| `canvas` | HTMLCanvasElement для отрисовки |
| `config` | Конфигурация игры (размеры, цвета, FPS) |

**Методы:**

- `render()` — запускает бесконечный цикл отрисовки.
- `addEffect(effect)` — регистрирует временный объект (пуля, взрыв) для отрисовки поверх поля.
- `destroy()` — очищает ресурсы и удаляет слушатели событий.

### GameController

Класс-диспетчер. Связывает пользовательский ввод, логику ИИ и обновление состояния.

**Аргументы конструктора:**
| Имя | Описание |
| :--- | :--- |
| `playerBoard` | Экземпляр доски игрока |
| `enemyBoard` | Экземпляр доски врага |
| `game` | Ссылка на основной класс Game |

**Ключевые методы:**

- `handleMouseDown(e)` — обрабатывает клики, делегируя их в `PhaseHandlers`.
- `playerHandler(cell)` — логика игрока (анимация -> расчет -> Store).
- `enemyHandler()` — асинхронная логика хода ИИ с использованием `await`.

## Графические объекты

### Board

Отвечает за отрисовку игрового поля 10x10.

| Метод                 | Описание                                                               |
| :-------------------- | :--------------------------------------------------------------------- |
| `render(boardData)`   | Отрисовывает сетку и состояние ячеек (корабль, промах, попадание)      |
| `getCellCenter(x, y)` | Возвращает экранные координаты центра ячейки для прицеливания эффектов |

### BaseElement (Абстрактный класс)

Базовый класс для всех анимированных объектов. Содержит метод `draw()`, который вызывается в лупе Game кажный кадр.

## Искусственный интеллект (EnemyAI)

Класс инкапсулирует стратегии стрельбы.

**Методы:**

- `getNextShot()` — возвращает координаты для следующего выстрела.

---

## Логика игры (Utils)

Чистые функции, вынесенные из классов для тестируемости:

- **fireShot(coords, board)**:
  Принимает координаты и текущую доску. Возвращает новую копию доски и результат (`hit`, `miss`, `killed`).
- **validatePlacement(board, x, y, size, direction)**:
  Проверяет возможность установки корабля с учетом "правила соседа".

## Состояние (Store)

Реализовано на базе `EventBus`. На данный момент EventBus не используется
луп игры сам подхватывает свежие данные для отрисовки

```typescript
interface GameStore {
  phase: 'SETUP' | 'BATTLE' | 'RESULT'
  playerBoard: cellType[][]
  enemyBoard: cellType[][]
  currentTurn: 'PLAYER' | 'ENEMY'
}
```
